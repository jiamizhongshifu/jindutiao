## 🚀 如何使用模板

### 方法一: 直接复制替换 (推荐)

1. **选择模板**:
   - 找到适合你的模板文件

2. **备份当前任务**:
   ```bash
   复制 tasks.json 为 tasks_backup.json
   ```

3. **应用模板**:
   ```bash
   # 复制模板文件
   copy tasks_template_24h.json tasks.json

   # 或者
   copy tasks_template_workday.json tasks.json

   # 或者
   copy tasks_template_student.json tasks.json
   ```

4. **自动生效**:
   - 如果 PyDayBar 正在运行,会自动重载
   - 如果没运行,直接启动即可

### 方法二: 通过配置界面导入

1. **打开配置工具**:
   - 双击 `PyDayBar-Config.exe`
   - 或右键托盘 → ⚙️ 打开配置

2. **清空现有任务**:
   - 删除所有现有任务 (点击 🗑️ 按钮)

3. **手动添加模板任务**:
   - 点击 "➕ 添加任务"
   - 参考模板文件手动输入

4. **保存设置**:
   - 点击 "保存所有设置"

## ✏️ 自定义模板

### 修改时间

```json
{
    "start": "09:00",  // 改为你的开始时间
    "end": "12:00",    // 改为你的结束时间
    "task": "上午工作",
    "color": "#4CAF50"
}
```

### 修改任务名称

```json
{
    "start": "09:00",
    "end": "12:00",
    "task": "写代码",  // 改为你的任务名
    "color": "#4CAF50"
}
```

### 修改颜色

```json
{
    "start": "09:00",
    "end": "12:00",
    "task": "上午工作",
    "color": "#FF5722"  // 改为你喜欢的颜色
}
```

## 💡 使用技巧

### 技巧 1: 多模板切换
为不同场景准备多个模板:
```
tasks_weekday.json   (工作日)
tasks_weekend.json   (周末)
tasks_vacation.json  (假期)
```

快速切换:
```bash
copy tasks_weekend.json tasks.json
```

### 技巧 2: 细化时间颗粒度
对重要时段进行细分:
```json
[
    {"start": "09:00", "end": "09:30", "task": "回顾邮件", "color": "#4CAF50"},
    {"start": "09:30", "end": "11:00", "task": "专注编码", "color": "#2196F3"},
    {"start": "11:00", "end": "12:00", "task": "代码审查", "color": "#00BCD4"}
]
```

### 技巧 3: 使用相似颜色分组
同类任务使用相近颜色:
```json
工作相关: #4CAF50, #66BB6A, #81C784 (绿色系)
休息相关: #FFC107, #FFD54F, #FFEB3B (黄色系)
学习相关: #9C27B0, #BA68C8, #CE93D8 (紫色系)
```

### 技巧 4: 留白时间
不要填满所有时间,预留弹性:
```json
// 09:00-09:30 没有任务,留作缓冲
{"start": "09:30", "end": "12:00", "task": "工作", "color": "#4CAF50"}
```

## ⚠️ 注意事项

### 时间格式
- ✅ 正确: `"09:00"`, `"14:30"`
- ❌ 错误: `"9:00"`, `"2:30 PM"`

### 时间顺序
- 结束时间必须晚于开始时间
- 可以跨越午夜: `"23:00"` 到 `"24:00"`

### 颜色格式
- ✅ 正确: `"#4CAF50"` (6位十六进制)
- ❌ 错误: `"#4CF"`, `"green"`

### 文件编码
- 使用 UTF-8 编码保存
- 确保 JSON 格式正确

## 🆘 常见问题

### Q: 模板不生效?
A: 检查以下几点:
1. JSON 格式是否正确 (用 JSONLint 验证)
2. 文件名是否为 `tasks.json`
3. 手动重载: 托盘 → 🔄 重载配置

### Q: 时间重叠怎么办?
A: 后面的任务会覆盖前面的,建议避免重叠

### Q: 任务太多显示不全?
A: 减少任务数量,或合并相似时段

### Q: 如何恢复默认?
A: 删除 `tasks.json`,重启程序会自动创建默认任务
