## 脚本结构（6场景式 + 小红书笔记）

每期漫画固定 6 个场景，Nana Banana Pro 会将每个场景渲染为独立格子：

| 场景 | 内容 | 说明 |
|------|------|------|
| 场景1-4 | 剧情 | 轻松搞笑故事（60%日常 + 25%技术 + 15%情感） |
| 场景5 | Gaiya 小课堂 | 1个开发/设计/效率知识点 |
| 场景6 | 产品引导 | 产品名 + Slogan + 下载链接 |

**小红书笔记**：
- 标题：15字以内，吸引眼球，可带emoji
- 正文：开头hook + 故事/知识点 + 产品引导 + 固定签名 + 标签

**示例结构**：
```
场景1：引入话题（钟师傅在做什么）
场景2：Gaiya 提问或吐槽
场景3：展示问题/冲突
场景4：解决或反转（笑点）
场景5：Gaiya 小课堂（知识点）
场景6：产品引导（固定模板）
+ 小红书笔记（标题 + 正文 + 标签）
```

## 每日工作流程

```
1. 开发工作完成
2. 告诉 Claude "生成今天的漫画脚本"
3. Claude 总结今日素材，生成脚本初稿
4. 修改确认后保存到 comics/scripts/
5. 喂给 Nana Banana Pro 生成漫画
6. 发布到小红书
```

---

## 快速命令

在 Claude Code 中运行：

```
请根据今天的开发工作，生成漫画脚本初稿
```

或指定主题：

```
请生成一期关于 [主题] 的漫画脚本
```
