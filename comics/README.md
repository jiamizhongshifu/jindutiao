## 脚本结构（Page式 + 小红书笔记）

### 🎯 创作核心理念

**每期漫画的主线必须是：产品问题 → 设计思考 → 心得分享**

1. **从真实开发问题出发**：今天遇到了什么产品/设计问题？
2. **展示思考过程**：为什么这是个问题？用户视角是什么？
3. **分享设计心得**：总结出可复用的产品设计原则或用户体验经验

漫画不只是记录开发日常，更是**产品设计思考的可视化表达**。

---

### 📄 Page 结构

以 Page 为单位生成脚本，给 Nana Banana Pro 更多创作自由度：

| 内容模块 | 说明 |
|----------|------|
| 问题引入 | 钟师傅遇到的产品/设计问题 |
| 思考与解决 | Gaiya 的启发、思考过程、解决方案 |
| 成果展示 + 知识点 | 改进效果 + Gaiya 小课堂（产品设计原则） |
| 产品引导 | 通过对话自然带出，不出现logo/下载链接（避免硬广） |

**注意**：
- 页面总数应合理分配内容，既不过于拥挤也不过于分散
- 确保每页都有明确的学习目标
- 每个 Page 只提供剧情概要和参考对话，不细分场景格子，由 Nana Banana Pro 自由排版

**小红书笔记**：
- 标题：15字以内，吸引眼球，可带emoji
- 正文：开头hook + 故事/知识点 + **产品设计心得** + 产品引导 + 固定签名 + 标签

**⚠️ 产品设计心得要求**：

小红书笔记的核心价值在于分享产品设计经验和思考，需要包含：

- 设计决策背后的思考过程（为什么这样做）
- 用户体验优化的具体技巧
- 可复用的产品设计原则（如"少即是多"）
- 实际数据或案例佐证（如"每多一个弹窗，跳出率涨5%"）
- 篇幅建议：产品设计心得应占正文的 40-50%

**示例结构**：

```text
Page 1：问题引入（钟师傅遇到什么问题，用户痛点）
Page 2：思考与解决（Gaiya 启发、解决过程）
Page 3：成果展示 + Gaiya 小课堂（产品设计原则总结）
Page 4/5：温馨收尾 + 软性产品引导（通过对话自然带出）
+ 小红书笔记（标题 + 产品设计心得 + 标签 + 产品信息）
```

## 每日工作流程

```
1. 开发工作完成
2. 告诉 Claude "生成今天的漫画脚本"
3. Claude 总结今日素材，生成脚本初稿
4. 修改确认后保存到 comics/scripts/
5. 喂给 Nana Banana Pro 生成漫画
6. 发布到小红书
```

---

## 快速命令

在 Claude Code 中运行：
