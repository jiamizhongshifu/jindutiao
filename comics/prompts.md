# 漫画脚本生成提示词

> 漫画脚本生成需要按顺序使用以下 2 个提示词

---

## Step 1: 代码分析提示词

**用途**：分析今日代码改动和开发日志，提取产品设计心得

```
你是一位专业的产品设计专家。请仔细阅读这段开发日志/功能实现，并完成以下任务：

1. 核心功能总结：用通俗易懂的语言概括代码解决的问题、实现思路和最终效果
2. 设计决策提取：识别代码中体现的产品设计决策、用户体验考量，以及"为什么这样做"的思考
3. 问题-解决链梳理：梳理从"遇到什么问题"到"如何解决"的逻辑链条
4. 设计心得提炼：提取可复用的产品设计原则或用户体验经验（如"少即是多"、"替用户做决定"等）
5.故事化潜力：思考如何将这些内容转化为生动有趣的视觉故事

输出要求：
- 语言通俗易懂，避免过度技术化的表述
- 重点突出产品设计层面的思考，而非纯技术实现细节
- 挖掘代码背后的用户体验洞察
- 为后续的漫画创作提供清晰的故事框架

请以钟师傅（GaiYa开发者，认真但偶尔犯傻）和Gaiya（时间女神，吐槽担当+产品设计启发者）为主角，设想他们如何以轻松有趣的方式向读者讲解这个功能背后的产品设计心得。

```

---

## Step 2: 故事规划提示词

**用途**：基于代码分析结果和开发日志，设计漫画学习读本的页面规划方案。

```
基于上述代码分析结果和开发日志，请设计一个漫画学习读本的页面规划方案。

角色设定：
- 主角：钟师傅（GaiYa 开发者，代表独立开发者视角，认真但偶尔犯傻，深夜肝代码）
- 导师：Gaiya（时间女神，产品吉祥物，吐槽担当 + 产品设计启发者，古希腊风格）

页面设计原则：
1. 循序渐进：从简单到复杂，符合认知规律
2. 场景化呈现：每页都有具体的场景（如教室、实验室、户外等）
3. 互动性强：展现钟师傅提问、GaiYa解答的互动过程
4. 视觉化重点：关键概念用道具、实验、类比等视觉元素呈现

每页内容要求：
- description: 该页的叙事内容，包含对话要点和知识点
- visualCue: 详细的视觉场景描述，包括场景环境、角色动作表情、关键道具、画面构图建议

注意：页面总数应该合理分配内容，既不过于拥挤也不过于分散，确保每页都有明确的学习目标。

小红书笔记要求：
- 标题：15字以内，吸引眼球，可带emoji
- 正文：开头hook + 故事/知识点 + 产品设计心得（占40-50%） + 产品引导 + 标签
- 设计心得需包含：决策思考过程、用户体验技巧、可复用原则、数据/案例佐证

产品信息（固定）：
- 产品名：GaiYa每日进度条
- Slogan：用进度条，让时间流逝清晰可见
- 官网：gaiyatime.com

```

---

## 脚本生成流程（Step 1 → Step 2）

```text
1. 开发工作完成后，整理今日代码改动
2. 使用【Step 1: 代码分析提示词】分析代码，提取产品设计心得
3. 使用【Step 2: 故事规划提示词】生成漫画脚本（Page 1-4 + 小红书笔记）
4. 保存脚本到 comics/scripts/XXX_主题.md
5. 发布到小红书
```

## Step 3: 图片生成提示词（独立使用）

**用途**：配合 Nana Banana Pro 或其他 AI 绘图工具生成漫画图片，独立于脚本生成流程

```
请生成一幅日式漫画风格的学习插图。
画面要求：
- 风格：温馨友好的日式漫画风格
- 主角：参考图1为钟师傅的形象, 参考图2为GaiYa的形象
- 色彩：明亮温暖，适合学习场景
- 细节：清晰的表情和动作，确保知识点的视觉化表达
画面构图：
- 人物比例协调，表情生动
- 背景简洁但能体现场景氛围
- 重点内容通过对话框、标注或特效突出
- 适合阅读的文字区域（如对话框）
请确保画面既有趣味性又有教育性，让读者能够轻松理解所传达的知识点。
页面分辨率统一为竖屏 2:3，语言是中文。

```
